# Makoto詞書 — 产品需求文档

## 概述

**Makoto詞書** 是一款 iOS 背单词应用，采用原生 Swift + SwiftUI 开发。

与市面上大多数背单词 app 不同，Makoto詞書 的核心理念是**像刷短视频一样刷单词**——通过简单直观的上下滑动交互，让背单词变得轻快、无压力、容易坚持。

---

## UI 设计

### 1. 首页

极简布局，纵向排列以下信息：

- **总词数量**：当前词书中的单词总数
- **已刷数量**：至少刷过 1 次的单词数
- **已刷轮次**：完成整轮刷词的次数（一轮 = 所有单词至少被刷到一次）
- **本轮进度条**：显示当前轮次的完成进度

页面底部放置醒目「Start Reviewing」按钮，点击进入刷词页。页面右上角放置**齿轮图标**，点击进入设置页。

> 整个 app 无需底部导航栏（TabBar），仅靠首页的两个入口完成所有操作。

### 2. 刷词页

- 单张单词卡片全屏展示
- 卡片内容简洁：**英文单词** + **中文释义**，不做多余装饰
- 纯手势操作，无按钮干扰
- 右上角关闭按钮（X）

### 3. 设置页

通过首页齿轮图标进入：

- **词书管理**：查看、切换、删除已创建的词书，每个词书有独立的进度数据
- **导入单词**：选择 `.txt` 文件 → 预览解析结果 → 命名词书 → 确认导入。每次导入自动创建新词书
- **清空当前词书**：一键清空当前词书的单词和进度（需二次确认）
- **关于**：app 版本信息

## 核心功能

### 1. 刷单词

交互方式模拟短视频滑动体验：

| 手势 | 行为 |
|------|------|
| **向上滑动** | 下一个单词 |
| **向下滑动** | 上一个单词 |

- 单词按导入顺序排列
- 瞬间切换 — 无动画延迟，每次只渲染一张卡片，无论词库多大都保持流畅
- 进入刷词页时，自动定位到上次退出时的单词位置（断点续刷）
- 刷完所有单词后自动返回首页
- 每刷过一个新单词，更新首页的统计数据

#### 刷词进度判定规则

- 一个单词只要向前刷过去了，即视为"已刷到"
- 当所有单词都被刷到过一次时，轮次 +1，本轮已刷数归零进入下一轮

### 2. 多词书管理

- 每次导入 txt 文件时自动创建新词书，导入前可为词书命名
- 在词书之间自由切换，每个词书保留独立的刷词进度和统计数据
- 支持左滑删除词书（删除时一并清除该词书的单词和进度）
- 活跃词书 ID 持久化存储，重启 app 后保持选择

### 3. 导入单词

- 支持读取 `.txt` 文件
- 文件格式要求：

  ```
  apple 苹果
  book 书
  cat 猫
  ```

  - 每行一个单词
  - **英文在前，中文在后**，中间以一个空格分隔
  - 空行自动忽略

- 重复导入时的处理：
  - 若英文单词已存在 → 更新其中文释义
  - 若英文单词不存在 → 追加到当前词书
  - 不删除已有单词（除非用户手动清空）

## 数据持久化

- 使用 SwiftData 本地存储所有数据
- 数据模型：

  | 模型 | 字段 | 说明 |
  |------|------|------|
  | `WordBook` | `id`, `name`, `createdAt` | 词书，一个词书包含多个单词 |
  | `Word` | `english`（唯一）, `chinese`, `lastSeenAt`, `createdAt`, `wordBookID` | 单词，归属某个词书 |
  | `ProgressState` | `bookID`, `currentIndex`, `completedRounds`, `reviewedEnglishWords` | 每个词书一个进度记录 |

- 刷词进度（当前位置、已刷轮次）按词书独立持久化，重启 app 后不丢失
- 活跃词书选择存储在 `@AppStorage` 中

---

## 非功能需求

- **最低支持 iOS 18+**
- 仅支持**竖屏**模式
- 支持**深色模式**
- 无需网络连接，完全离线可用
- 单卡片渲染 — 无论词库多大，刷词性能始终流畅
- 配置 Launch Screen 确保全面屏设备正确显示
